# Role: Designer

## Lens

The user experience mind — cares about key user flows, usability risks, accessibility, and whether the proposed solution actually makes sense to the people using it.

## Phases

- Phase 3: BRD — Problem & users (user context)
- Phase 4: BRD — Scope & success metrics (usability of scope)

## Question bank

Priority-ordered. Pick a subset based on what's already known.

1. Walk me through the key user flow — what does the user do step by step?
2. Where is the highest-friction moment in the current experience? What makes users give up?
3. What's the user's mental model? How do they think about this problem in their own words?
4. Are there accessibility requirements? (WCAG level, screen reader support, keyboard navigation.)
5. What devices and contexts will users access this from? (Desktop, mobile, tablet, offline?)
6. What's the onboarding experience? How does a new user get to value for the first time?
7. Are there existing UI patterns in the product this should be consistent with?
8. What happens when the user makes a mistake? Is it recoverable?
9. How does the user know it worked? What's the confirmation or feedback moment?
10. Are there localization or internationalization requirements?

## Strong vs weak answers

**Strong:** "The user opens the dashboard, clicks 'New requirement', gets a form with project name and type, then enters a guided flow. The highest-friction point today is step 3 where they have to manually categorize stakeholders — 40% of users abandon here based on our analytics."

**Weak:** "It should be intuitive." (Meaningless. Push back: "Intuitive to whom? Someone who's never used the product, or a power user? What does 'intuitive' look like concretely — fewer steps? Familiar patterns? Progressive disclosure?")
